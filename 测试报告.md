## 已实现的核心模块

### 原有模块
- `indicator_agent.py` - 指标智能体（RSI, MACD, Stoch, ROC, Williams %R）
- `pattern_agent.py` - 模式智能体（16种K线形态识别）
- `trend_agent.py` - 趋势智能体（趋势线拟合、支撑阻力分析）
- `decision_agent.py` - 决策智能体（综合判断 LONG/SHORT）
- `trading_graph.py` - 主交易图编排（LangGraph）
- `web_interface.py` - Flask Web 界面
- `default_config.py` - 默认配置
- `static_util.py` - 静态工具（图表生成等）
- `graph_util.py` - 技术指标工具（TA-Lib 封装）
- `graph_setup.py` - LangGraph 构建
- `agent_state.py` - LangGraph 状态定义

### 新增模块（本次实现）
| 文件 | 职责 | 说明 |
|------|------|------|
| `data_providers/base.py` | 抽象基类 | `BaseDataProvider` + `FetchRequest` + 规范化/验证管道 + 自定义异常 |
| `data_providers/yahoo.py` | Yahoo Finance | 从 yfinance 获取数据，处理 MultiIndex 列名 |
| `data_providers/qlib.py` | Qlib 本地数据 | 修复 .bin 读取（原代码错误地查找 .pkl），日历对齐逻辑 |
| `data_providers/akshare.py` | AKShare（新增） | 支持美股、国际期货、全球指数，完全免费 |
| `data_providers/registry.py` | 注册表 | `ProviderRegistry` + `get_provider()` 工厂函数 |
| `data_providers/__init__.py` | 公共 API | 导出所有 Provider 类和异常 |

## 已知问题

### 1. LLM 配额限制
- **问题**: DashScope API 配额耗尽，LLM 调用超时
- **影响**: Pattern Agent 和后续分析无法完成
- **状态**: ⚠️ 等待配额恢复后重试
- **临时方案**: 减少请求频率或更换 API key

### 2. 视觉模型配置
- **问题**: `graph_llm_model` 当前为 `qwen3.6-plus`，不支持图像输入
- **影响**: Pattern Agent 和 Trend Agent 的视觉分析功能不可用
- **修复建议**: 将 `graph_llm_model` 改为 `qwen3-vl-plus`

### 3. 期货数据接口偶发异常
- **问题**: AKShare `futures_foreign_hist` 偶尔返回无效 JSON
- **影响**: GC/CL 期货数据获取不稳定
- **状态**: 已有错误处理，不影响其他资产
