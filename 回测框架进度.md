## 一、背景

原有 QuantAgent 框架只能做单点分析（给一段 K 线 → 输出做多/做空建议），缺少历史回测验证环节。需要新增独立回测模块来验证多智能体系统的实际预测价值。

## 二、架构决策

| 决策项 | 选择 | 原因 |
|---|---|---|
| 回测框架 | **Backtrader**（成熟框架） | 不手写回测逻辑，利用成熟的订单管理/手续费/滑点/分析器 |
| 对原框架 | **零侵入** | 新代码全部在 `backtesting/` 目录，不改动现有文件 |
| 运行模式 | **双模式** (fast/full) | Fast 用于参数调优，Full 用于最终验证 |
| 做空处理 | A 股指数不能做空 | LONG = 买入/持有，SHORT = 清仓/空仓 |
| 成交价格 | **按收盘价** | 止盈止损按 close 判断，冲突时优先止损（保守） |
| 数据源 | AKShare A 股指数日线 | 创业板指/沪深300/中证1000/北证50/科创50 |
| 过程数据存档 | **按股票代码+时间戳分目录** | 避免 record.csv 被覆盖，保留每次运行的 K 线图和数据快照 |

## 四、数据流概览

```
AKShare 指数数据 (DataFrame, ~78-295 bars)
    │
    ▼
_df_to_bt_data() → Backtrader PandasData Feed
    │
    ▼
Cerebro.addstrategy(QuantAgentStrategy, trading_graph=...)
    │
    ▼  每个 bar 调用 next()
    │
    ├─ 无持仓 → _get_window_data() → 最近 45 根 K 线 dict
    │            │  (使用负索引，暖窗期间自动截断)
    │            ▼
    │         trading_graph.graph.invoke(initial_state)
    │            ▼
    │         Indicator → Pattern → Trend → Decision
    │            ▼
    │         解析 JSON → 决策/止盈/止损
    │            ▼
    │         self.buy()  ← Backtrader 执行买入
    │            ▼
    │         self.broker.getvalue() → 记录到 broker_values
    │
    └─ 有持仓 → 检查 close vs stop_loss/take_profit
                ↓
             self.close()  ← Backtrader 执行平仓

过程数据存档:
  graph_util.generate_kline_image() → backtest_data/{symbol}/{timestamp}/kline_chart.png + record.csv
  static_util.generate_trend_image() → backtest_data/{symbol}/{timestamp}/trend_graph.png
```

## 九、注意事项

- **LLM API 配置** 在 `default_config.py` 中，代理地址 `base_url: https://new.samerwang.dpdns.org`
- **代理稳定性** 是主要瓶颈，524 超时频繁出现，建议增加重试机制或使用更稳定的代理
- **Fast 模式** 需要 LLM 调用（Indicator + Decision Agent），每次调用可能需要数秒
- **Full 模式** 额外需要视觉 LLM 分析 K 线图，每次调用更慢
- **LLM 缓存** 首次运行后会保存到 `.backtest_cache/`，后续相同窗口不会重复调用
- **AKShare 数据** 需要联网获取，部分指数可能数据有限
- **过程数据** 保存在 `backtest_data/{symbol}/{timestamp}/` 目录下，包含 record.csv、kline_chart.png、trend_graph.png
